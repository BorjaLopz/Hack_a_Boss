Otro tipo de formulario

Como todos los elementos de un formulario son elementos de linea, podremos hacer un lista de campos y hacemos una lista donde cada accion con su label y con su input

  • Atributos minimo para valor minimo, maximo para valor maximo y step para ver cada cuanto queremos aumentar la cantidad

  • valor del slider
  • Este tipo de atributo no tiene mucho sentido, ya que poner algo para que no se vea no nos sera util
  • En este caso no podremos modificar el color con CSS mas que su tamaño. Si quisieramos tener un checkbox personalizado, tendremos que ocultar esto y visibilizar el nuestro.
  • En caso de que nos parezca feo el boton, podemos ocultar el input (lo que pone choose file) y dejar el label que funcione como ese.

    Ademas podremos delimitar que archivos queremos que acepte y cuales no con el atributo accept

  • Existen 3 tipos de botones
    • Este enviara por defecto el formulario
    • Este por defecto reseteara el formulario
    • Este no hace nada, por defecto se usa fuera del formulario

Atributos de campos de un formulario

  • id describe el label;mientras que name describe el campo

    Es importante que todas las cosas de un formulario tengan name ya que solo asi se enviaran los datos Sino tiene el atributo name no se enviara nada

  • a

  • Esto lo que hace es que cuando abramos la pagina web directamente podamos escribir ahi

    Solo podremos poner un autofocus solamente a una casilla

  • Con este atributo lo que hacemos es que un campo sea necesario para enviarlo

  • Esto ns servira para poner un texto de una longitud minima de X y una maxima de Y

  • Esto se usara en un futuro con JS para cuando no hayamos metido el usuario, no podremos meter el correo

    Otro ejemplo, es una casilla de provincia pero previamente seleccionamos el pais. Si no es España lo deshabilitamos

    Ademas, todo lo que sea disabled no se enviara en el formulario.
